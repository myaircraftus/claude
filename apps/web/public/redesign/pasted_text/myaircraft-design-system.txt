NON-NEGOTIABLE PRODUCT TRUTHS
-----------------------------------

The UI must visually respect these truths:

- The platform is **chat-first**, but chat is only one part of the system.
- Answers are **source-backed and citation-based**.
- Aircraft-by-aircraft organization is core.
- Different user roles matter.
- Permissioning and signature authority matter.
- Signed maintenance actions should feel controlled, professional, and not casually editable.
- RAG / retrieval remains the source of truth.
- Behavior/personalization improves defaults and workflow relevance, but does not replace records truth.
- The product flow is:
  ingestion -> normalization -> structured storage -> retrieval -> orchestration -> response

The design must make all of this feel obvious and premium.
